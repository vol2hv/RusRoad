Задача. Система сбора информации из файлов.
Сущности базы данных:
1. Автодорога. Поля – наименование, предельная скорость passage.txtдвижения
2. Автовладелец. Поля – ФИО, гос. номер автомобиля.
3. Проезд. Поля – идентификатор дороги, идентификатор автовладельца, скорость автомобиля, время фиксации

Необходимо:
Разработать структуру БД и описать создание в виде sql-скрипта. 
Разработать windows-сервис сбора информации о передвигающихся по дорогам автомобилях и их скорости. 
В разрабатываемую систему от различных датчиков фиксации скорости приходит информация в виде csv-файлов, содержащих:
гос. номер автомобиля, 
идентификатор автодороги, 
время фиксации, 
скорость, с которой двигался автомобиль. 
Сервис должен загружать эти данные в БД по мере их поступления. 
В случае нахождения ошибок в пришедших данных (например, указанный номер отсутствует в таблице автовладельцев ли неверный 
идентификатор дороги) ошибочные записи не должны загружаться в БД, но должны записаться без изменений в отдельный файл 
в специальной директории и в отдельный файл должны быть записаны все ошибки, исходя из которых записи признаны ошибочными.
Каждые 8 часов сервис должен делать выгрузку отчета по автовладельцам, превысившим предельную скорость с указанием ФИО, 
номера автомобиля, наименования дороги и времени проезда в файл формата csv. 
Также для каждого такого водителя должен быть сформирован файл с персональным уведомлением, содержащий текст-обращение
 с указанием ФИО водителя, информацией о том, что он нарушил скоростной режим с указанием времени, дороги где это произошло, 
 предельной скорости и фактической скорости водителя, а также предложение явиться в ближайшее отделение ГИБДД для оплаты штрафа.

Каталоги и файлы
\RusRoadClient	клиет для сервера и сервиса (передает через TCP файл Passage.txt)
\RusRoadLib		библиотека функционала проекта. Весь функционал здесь. Сервер и сервис только обертки для встаивание в windows. 
\RusRoadServer	сервер. Служит для отладки, которая была бы затруднена непосредственно при оформлении процесса как сервиса.
\RusRoadService	сервис.Функционал встроенный в оболочку windows-сервисов.

CreateDB.sql Запрос на создание БД и заполнение ее справочников

RusRoads.mdf БД проекта. При ее наличии в корневом каталоге проекта сервер и сервис работают с ней. 
(Экземпляр сервера SQL(LocalDB)\MSSQLLocalDB). 
Если эта БД отсутствует берется БД согласно строке подключения из App.config

passage.txt Записи проездов для тестирования

Сервис отчетов для удобства тестирования стартует каждые 5 минут. Для изменения - подправить  RoadsReport.CountHandler.
Граница обработанных сервером отчетов данных хранится в БД RusRoadCommon.LastReport.
